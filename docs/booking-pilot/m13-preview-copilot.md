# M13 — Preview Copilot: canonical scripts

## Files

- `lib/relay/preview/booking-scripts.ts` (new)
- `src/app/admin/relay/health/preview/` (new panel component under the Health page)

## What to build

### Scripts (data, not prompts)

Author **~8 scripted customer scenarios per booking sub-vertical**:

- hotel
- clinic
- wellness
- ticketing
- airport_transfer

Total: ~40 scripts.

Each script is a plain sequence:

```ts
{ role: 'user', content: string }[]
```

No AI. No "generated by LLM". Hand-written data.

### Preview panel

- Partner selects a script → panel runs it against the live orchestrator with the partner's current config.
- Each turn renders inline: block used, text, suggestions.
- **Reproducibility:** same script + unchanged config → identical path. If the path differs turn-to-turn with unchanged config, that's a bug to chase before closing the milestone.

## Constraints

- No new AI calls anywhere in the preview path.
- Do not add any randomness to the orchestrator to support preview — if randomness exists today, seed it from `session.turnId` so previews stay reproducible.

## Acceptance

- All 40 scripts run without orchestrator errors.
- Panel accessible from `/admin/relay/health` for any partner with Booking engine enabled.
- Reproducibility check: running the same script twice on unchanged config produces identical block/text/suggestions per turn.
